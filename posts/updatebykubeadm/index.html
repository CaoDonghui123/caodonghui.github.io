使用kubeadm升级集群

2021.02.15

1.升级kubeadm 版本(master)

## 升级kubeadm
$ yum update -y kubeadm-1.20.2-0

## 检查可升级到哪些版本，并验证您当前的集群是否可升级
$ kubeadm upgrade plan

查看输出日志可以得知可升级的版本

2.查看所需镜像(master)

$ kubeadm config images list

k8s.gcr.io/kube-apiserver:v1.20.2
k8s.gcr.io/kube-controller-manager:v1.20.2
k8s.gcr.io/kube-scheduler:v1.20.2
k8s.gcr.io/kube-proxy:v1.20.2
k8s.gcr.io/pause:3.2
k8s.gcr.io/etcd:3.4.13-0
k8s.gcr.io/coredns:1.7.0

3.拉取镜像 脚本

## vi 可以直接粘贴进去，vim会有点点问题
$ vi pull-image.sh

使用的是containerd

## 设置镜像仓库地址
MY_REGISTRY=registry.cn-hangzhou.aliyuncs.com/google_containers

## 拉取镜像
ctr image pull ${MY_REGISTRY}/kube-apiserver:v1.20.2
ctr image pull ${MY_REGISTRY}/kube-controller-manager:v1.20.2
ctr image pull ${MY_REGISTRY}/kube-scheduler:v1.20.2
ctr image pull ${MY_REGISTRY}/kube-proxy:v1.20.2
ctr image pull ${MY_REGISTRY}/etcd:3.4.13-0
ctr image pull ${MY_REGISTRY}/pause:3.2
ctr image pull ${MY_REGISTRY}/coredns:1.7.0
## 设置标签
ctr image tag ${MY_REGISTRY}/kube-apiserver:v1.20.2          k8s.gcr.io/kube-apiserver:v1.20.2
ctr image tag ${MY_REGISTRY}/kube-scheduler:v1.20.2          k8s.gcr.io/kube-scheduler:v1.20.2
ctr image tag ${MY_REGISTRY}/kube-controller-manager:v1.20.2 k8s.gcr.io/kube-controller-manager:v1.20.2
ctr image tag ${MY_REGISTRY}/kube-proxy:v1.20.2              k8s.gcr.io/kube-proxy:v1.20.2
ctr image tag ${MY_REGISTRY}/etcd:3.4.13-0                    k8s.gcr.io/etcd:3.4.13-0
ctr image tag ${MY_REGISTRY}/pause:3.2                       k8s.gcr.io/pause:3.2
ctr image tag ${MY_REGISTRY}/coredns:1.7.0                   k8s.gcr.io/coredns:1.7.0

$ su pull-image.sh

4.升级集群(master)

$ kubeadm upgrade apply 1.20.2

5.升级kubelet kubectl (master)

## 升级 kubectl
$ yum update -y kubectl-1.20.2-0

## 升级 kubelet
$ yum update -y kubelet-1.20.2-0

## 重新加载系统配置
$ systemctl daemon-reload

## 重启 kubelet
$ systemctl restart kubelet

## 查看 kubelet
$ systemctl status kubelet

6.升级worker节点的kubeadm kubelet

## 设置节点为不可调度(master)
$ kubectl drain [nodename] --ignore-daemonsets

worker

## 升级 kubeadm
$ yum update -y kubeadm-1.20.2-0

## 升级 kubelet
$ yum update -y kubelet-1.20.2-0

## 重新加载系统配置
$ systemctl daemon-reload

## 重启 kubelet
$ systemctl restart kubelet

## 查看 kubelet
$ systemctl status kubelet


## 取消节点不可调度(master)
$ kubectl uncordon [nodename]
